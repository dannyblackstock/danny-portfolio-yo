Active Textbook is a Ruby on Rails e-reading web application developed by Evident Point, a software company based in Richmond, British Columbia. I worked on the project as the sole in-house User Interface Designer / Developer, where I was tasked with redesigning many key features of the reader and user dashboard, as well as designing the interfaces for new features.

Hotspot Creation

One of the first tasks I was assigned involved redesigning a feature that allows users to add media or other interactive elements – called “hotspots” – on to a page. Since the target audience for this feature was educators who may not have technical backgrounds, I tried to create a workflow that would be straightforward and easy to use.

After the user selects where they would like to add the media to their e-book, I divided the following process into two steps, in order to guide the user and make the differences between media types more clear. For the first step, I designed large, clickable icons for each type of hotspot. The second step allows the user to input a title and content of the hotspot.

Media Library

While redesigning the hotspot creation process, I also designed the interface for a file storage system, called the “media library.” This feature allows the user to upload and reuse files when creating image, audio, video, or attachment hotspots. I created a modular solution that works well in different sizes and on different devices, and allows users to filter, preview, delete, and select media items in all of these contexts.

The media library being used to select an image for a hotspot.

In the user’s dashboard, a side panel appears that can be used to view detailed information about each media item, as well as see where each item is used in a user's Active Textbooks.

The side panel in the user dashboard allows users to view detailed information about each media item.

Settings Modal

Another feature I designed for the Active Textbook reader was the settings modal window, which allows users to view and modify information about their book and media library. I decided to use a side tab bar to split up the information, in order to be more digestible by the users, and so that more tabs can be added in the future if necessary. I also wanted the tabs to be large enough to be easily clickable on touch devices.

I designed interfaces for native iOS, Android, and Windows applications for Active Textbook as well.
